动态换路： 允许在卡片上直接更改“执行渠道（API Key/BaseURL）”或“执行模型”，满足紧急切换低成本模型的需求。

3. 提示词 PAPI 系统 (Prompt AI-Profile Interface)
逻辑： 引入变量层，实现“任务-角色”的解耦。

变量定义： 在 Session 配置中定义功能变量：
${UI_AGENT} = Gemini-2.0-Flash (利用其多模态能力分析截屏)
${BACKEND_DEBUGGER} = Codex-O3 (利用其逻辑深度)
${DOC_CLEANER} = Qwen-2.5-7B (低成本快速处理)
动态注入：
当主 AI 调用工具 dispatch_task(category="frontend", ...) 时，PAPI 系统自动查找 ${UI_AGENT} 指向的配置。
优势： 用户只需在全局配置中修改变量值，即可一键将所有前端任务从 Gemini 切换到 Claude，无需逐个修改提示词。
上下文感知路由： PAPI 可根据 Code Flow 导图中的“技术栈记忆”自动切换变量（如检测到 Python 项目，${DEBUGGER} 自动映射到专门优化过 Python 的渠道）。
4. 增强的状态恢复逻辑 (Enhanced Rollback)
逻辑： 针对多 AI 协作的“深度一致性”回滚。

全栈快照包含：
Git 代码状态。
主 AI 状态： 包含已生成的 Plan 执行进度。
子 AI 状态： 针对 Coder AI 的 /rewind 命令调用。
回滚行为： 当用户在 UI 任务卡片上点击“重试”：
GUI 发送 SIGINT 给当前所有 AI 进程。
执行 git reset --hard 回滚到该卡片创建时的快照。
通过 Hook 强制更新主 AI 的 Context，删除失效的执行记录。
E. [Uncertainty & Risks]
