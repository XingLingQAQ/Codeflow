# App Shell 与阶段自适应工作台详细设计

> 状态：Active / 目标态设计（当前实现仍为 `ViewMode` 控制台）  
> 里程碑：M1（骨架）、M3（编码画布）、M4（想法/设计/Review 画布）、M6（Live Preview）  
> 纳入跟踪：2026-07-13（M0.6）

---

## 1. 布局哲学

VS Code 是文件中心，CodeFlow 是流程中心。借鉴其可停靠面板、Command Palette、状态栏与编辑器网格（仅编码阶段内），但顶层导航围绕 Flow 阶段组织，中央区是随阶段整体变形的 Stage Canvas。

## 2. 工作台布局

```
┌──────────────────────────────────────────────────────────────┐
│ 顶栏：项目切换 | Flow 进度条（七阶段，可点击跳转/查看产物）        │
├──────┬───────────────────────────────────────────┬───────────┤
│ Flow │           Stage Canvas（阶段画布）          │  Agent    │
│ Rail │   中央区随阶段整体变形                       │ Companion │
│      │                                           │  （常驻）  │
├──────┴───────────────────────────────────────────┴───────────┤
│ 底部 Panel：终端 | 工作流日志 | 审计 | 问题 | 守卫（阶段联动默认页）│
├──────────────────────────────────────────────────────────────┤
│ 状态栏：当前阶段 | 模型/热切换 | Token 预算 | 快照点 | 守卫灯 | 连接 │
└──────────────────────────────────────────────────────────────┘
```

- 布局引擎：dockview（可拖拽分屏/停靠）
- 每阶段一个布局预设，用户调整按阶段记忆；阶段切换用共享元素过渡动画
- Agent Companion 跨阶段常驻，自动切换该阶段的系统提示词与工具集（对接 Main/Coder/Sub 角色配置）；集成渐进式记忆披露（输入预检弹窗 + 预警指示灯）
- 全局 Cmd+K：命令 + 混合搜索（FTS/向量/图谱）+ 阶段跳转 + Agent 召唤

## 3. 七种阶段画布规格

| 画布 | 主区 | 辅助面板 | 关键交互 |
|---|---|---|---|
| 意图画布 | 富文本 + Agent 追问气泡流 | 素材篮 | 追问逐条采纳/忽略；完成生成 idea.md |
| 设计画布 | 文档编辑 + Mermaid 架构图双栏 | 方案对比卡 | 发起辩论按钮；ADR 模板插入 |
| 规划画布 | 任务看板（P0/P1/P2 列）+ 依赖图切换 | 影响分析（导入流） | 拖拽排序；Agent 分解结果逐项采纳 |
| 调研画布 | 中：多源结果流 | 左：检索计划树；右：证据篮 | 证据拖入篮；结论沉淀 SAMG 提示 |
| 编码画布 | Monaco 多 Tab/分屏 + 面包屑 + 符号大纲 | 文件树、AST 上下文构建器（勾选 + Token 预算饼图）、守卫面板、Live Preview | 任务卡 ↔ 变更双向关联；选中代码加入上下文/发起辩论 |
| Review 画布 | 变更集逐文件 Diff | Critic 评审意见流、辩论面板、测试结果 | 意见逐条处理；Debug 子模式回环编码 |
| 提交画布 | 变更集 → commit 分组建议 | 快照绑定状态、豁免清单、归档报告预览 | Agent 生成 message；推送/建 MR 可选 |

## 4. 后端 internal/workspace（M3）

- 文件树 API：list/read/watch（WS 推送变更），符号大纲复用 internal/ast
- Dev Server 进程管理：探测 package.json scripts、启动/停止、日志接管到底部 Panel
- 安全：路径白名单限制在项目根内；写操作一律经 guard

## 5. Live Preview 与检查器桥（M6）

### 5.1 承载

- Tauri：内嵌 WebView 独立进程；浏览器模式：iframe + 本地代理解决跨域
- HMR 原生生效实现"实时自动显示"；console/network 错误由桥捕获供 Agent 引用

### 5.2 圈选反馈

- 注入脚本实现悬停高亮（组件名+尺寸）、点选/框选生成元素上下文包：DOM 路径 + 计算样式 + 截图 + 源码映射（React DevTools hook / babel data-source → 定位 Button.tsx:23）
- 标注层：截图画框写字，多标注一次提交

### 5.3 双模式

- 对话模式：上下文包作为附件发给 Agent Companion，适合提问/讨论/报 Bug
- 编辑模式：Agent 生成最小 diff（限映射源文件范围）→ 就地预览 → 应用（过守卫）→ HMR 刷新 → 满意归入当前任务变更集 / 不满意影子区一键撤销；跨文件大改自动建议转正式任务，防止旁路失控
- 进阶：响应式断点切换、元素 ↔ SAMG 三元组联动、改前后截图视觉回归

## 6. 网页版与手机版接入点

- 网页版：同一 workbench 构建产物经 backend/internal/web embed 分发；断点 ≥1280 全功能、768~1280 折叠 Flow Rail、<768 引导使用手机伴侣端
- 手机版：不渲染 Stage Canvas，仅消费 Flow 进度/Gate/Agent 对话（见 frontend-experience.md 第 5 节）
