## 1. 背景与目标

### 1.1 产品目标（来自设计）

CodeFlow 从「会话/计划管理控制台」升级为 **流程中心（Flow-Centric）智能开发工作台**：

- 顶层信息架构围绕 Flow 阶段，而非文件树
- 原生工作流引擎是第一公民：想法 → 设计 → 规划 → 调研（可选）→ 编码 ⇄ Review/Debug → 提交
- 桌面（Tauri）为主端，网页 embed 分发，手机为远程伴侣端
- 插件可贡献阶段类型、Gate、守卫规则与核心能力替换点

### 1.2 本计划目标

1. **以真实代码为准**建立能力差距矩阵（不把未实现模块写成已实现）
2. 给出 **M0–M7 可执行任务拆分**、硬依赖、验收标准与风险
3. **解决前端主线矛盾**（template 默认 vs roadmap 删除 template / `apps/workbench`）
4. 明确后端硬化专项（快照 restore、floweng、DI、域收敛、schema-first、WS 总线）
5. 给出 **可立即开工的第一批 PR**，避免双前端与假 restore 继续扩散

### 1.3 非目标（本计划明确不做或后置）

- 不在 M0–M2 并行维护两套完整前端业务逻辑
- 不把 `internal/workflow` 观测层伪装成 Flow 执行引擎
- 不在 restore 仅 token 校验时宣称「原子快照已完成」
- 不在 M3 前强推 Live Preview / 手机伴侣端完整产品化
- 不一次性删除全部 Get/Set 全局单例（按域分批，保证可回滚）

### 2.5 Snapshot 实现细节（M2 硬依赖）

文件：`backend/internal/snapshot/state_provider.go`

- Capture：对 conversation/vector/graph 做 `digestState` → `prefix:sha256[:12]` 指纹字符串
- Restore：`validateStateToken` 只检查非空与前缀；**不恢复任何业务状态**
- Git restore：默认 no-op；环境变量开启后 hard reset

因此当前 API 可演示「创建/列表/restore 成功标志」，但 **不具备** floweng「阶段完成点回跳」所需一致性。

- 缺：多方 N 角色、每方 model/channel、flow/stage 外键、共识策略插件化

### 10.3 PR-4 Snapshot 真 restore 设计要点（接口先行）

**现状锚点**：`backend/internal/snapshot/state_provider.go`

| 操作 | 现状 | 目标（PR-4 骨架） |
|---|---|---|
| Capture conversation | `digestState("conversation", trace)` → 指纹串 | 持久化 **可恢复载荷**：inline JSON 或 `storage_ref` + digest |
| Capture vector | memory List digest | 版本指针或导出 blob ref |
| Capture graph | samg stats+export digest | export 落盘/落库 + digest |
| Capture git | 真实 hash | 保持；restore 受控 |
| Restore conversation/vector/graph | `validateStateToken` 仅前缀校验 | 按 kind 调用 agent/memory/samg **写回** |
| Restore git | 默认 no-op；`CODEFLOW_SNAPSHOT_ENABLE_GIT_RESTORE=true` 才 Reset | 保留 opt-in；增加执行锁与审计；floweng 调用走受控 API |

**建议状态串 schema（示意，落地时定 ADR）**

`json
{
  "schema_version": 1,
  "kind": "conversation|vector|graph|git",
  "digest": "sha256:...",
  "storage": "inline|blob",
  "payload": {},
  "blob_ref": "snapshots/<id>/<kind>.json"
}
`

**PR-4 最小完成定义**

1. StateProvider 契约支持「指纹兼容旧 token」+「新 schema 真恢复」  
2. **至少一条**真恢复链路单测绿（优先 conversation **或** graph，因 capture 已有数据源）  
3. 旧 digest-only 快照：restore 返回明确「不可恢复/只读校验」而非假成功（或 feature flag 控制）  
4. 不强制本 PR 打开 git hard reset  

**与 M2 关系**：floweng 阶段完成自动快照、阶段回跳依赖本能力；故 PR-4 可与 M1 UI 并行，但 **阻塞 PR-8 / M2b**。
